Setupの方法を記述します。 


# /============================================================================
#                                                                    Java & Ant
#                                                                    ==========
1. Setup JRE

    SunのPageよりJRE-5.0(以上)をDownloadし、Installして下さい。｛必須｝


2. Setup Ant

    Ant-1.6(以上)を以下のSiteよりDownloadし、Installして下さい。｛必須｝

        Apache Ant

    Downloadして解凍し、環境変数として{ANT_HOME}を追加します。
    さらに、環境変数{Path}に%ANT_HOME%\binを追加します。


# /============================================================================
#                                                                       DBFlute
#                                                                       =======
3. Locate DBFlute

    DownloadしたDBFluteを解凍して、任意の場所へ配置。

    ※以下のDirectoryに配置することを推奨します。

      ex) DBFlute-0.5.2の場合：
        /- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
        xxxproject
            |
            +- .settings
            |
            +- dbflute_[dbname]          // DBFluteのClientDirectory ※後述
            |     |
            |     |
            |     +- dfprop
            |     +- log
            |     +- output
            |     +- playsql
            |     +- schema
            |     +- _project.bat ☆
            |     +- build-[dbname].properties
            |     +- ...
            |
            +- mydbflute                 // DBFlute本体の格納場所
            |     |
            |     +- dbflute-0.5.2       // ☆ここ
            | 
            +- src/main/java
            +- src/main/resources
            +- ...
            ...
        - - - - - - - - - -/


# /============================================================================
#                                                      DBFlute Client Directory
#                                                      ========================
4. Locate client directory(containing build-properties and batch-files)

    4-1. %DBFLUTE_HOME%/etc/client_directory-templateのDirectoryに、実行Directory(Client Directory)の
         Templateがあります。任意のTemplateをCopyし、LocalPCの任意の場所に配置して下さい。

         ※以下のDirectoryに配置することを推奨します。

           ex) DBFlute-0.5.2の場合：
             /- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
             xxxproject
                 |
                 +- .settings
                 |
                 +- dbflute_[dbname]          // ☆ここ
                 |     |
                 |     |
                 |     +- dfprop
                 |     +- log
                 |     +- output
                 |     +- playsql
                 |     +- schema
                 |     +- _project.bat/.sh
                 |     +- build-[dbname].properties
                 |     +- ...
                 |
                 +- mydbflute                 // DBFlute本体の格納場所
                 |     |
                 |     +- dbflute-0.5.2
                 | 
                 +- src/main/java
                 +- src/main/resources
                 +- ...
                 ...
              - - - - - - - - - -/

          ※Templateは、以下の２つが存在します。(大抵の方にはminimumPropertiesを利用することを推奨)

              - fullProperties:
                  全てのPropertyが記述されています。
                  DBFluteをかなり使い慣れていて、最初からFULL機能をじっくり考えるために
                  一個一個のPropertyを精査していく場合は、これをお奨めします。

              - minimumProperties:
                  最低限のPropertyが記述されています。
                  とりあえず自動生成をしてみてその後徐々に機能を追加していく場合は、
                  これをお奨めします。｛DBFlute初心者の方はこれ｝
                  (準備されているAnt-Taskはfullと同様です)

    4-2. _project.bat/.shの「MY_PROJECT_NAME」変数の値を[dbname]に変更して下さい。

    4-3. build-minimumProperties.propertiesのFile名をbuild-[dbname].propertiesに変更して下さい。

    ★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★★
    4-4. _project.bat/.shの「DBFLUTE_HOME」変数の値を設定して下さい。
         (CommentOutされているのを外して下さい)

             ex) DBFluteの配置場所が「xxxproject/mydbflute/dbflute-0.5.2」の場合：
               /- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
               set DBFLUTE_HOME = ../mydbflute/dbflute-0.5.2
               - - - - - - - - - -/

# /============================================================================
#                                                                    Properties
#                                                                    ==========
5. Modify build-properties

    まず「自動生成する前に必ず設定すべきもの」だけを設定します。
    build-[dbflute].propertiesに「@FirstProperty」というMark付いているものが該当します。

    ＜主な最初に決めるべきProperty＞
    o torque.database = [database]
    o torque.targetLanguage = [Java or C#]
    o torque.isJavaNameOfTableSameAsDbName = [true or false]
    o torque.isJavaNameOfColumnSameAsDbName = [true or false]
    o torque.projectPrefix = [prefix of class name]
    o torque.packageBase = [package name]
    o torque.s2daoVersion = [s2dao version]
    o DatabaseInfoMap (./dfprop/databaseInfoMap.dfpropに記載)
        ex) Apache Derbyを利用する場合：
        /- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - 
        map:{
            ; driver   = org.apache.derby.jdbc.EmbeddedDriver
            ; url      = jdbc:derby:../src/main/webapp/WEB-INF/data-derby;create=true
            ; schema   = LDB
            ; user     = ldb
            ; password = ldb
        }
        - - - - - - - - - -/
 
    ※基本設定については   → Basic Properties
    ※JDBCの設定については → JDBC Setting

    _/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/
    jdbc.diconの設定にて、Component定義を以下のように差し替えてください。
    これにより、Paging検索周りの機能が有効になります。
    (C#版は'FetchNarrowingResultSetFactory'のみ)

    component class="org.seasar.extension.jdbc.impl.BasicStatementFactory"
    component class="org.seasar.extension.jdbc.impl.BasicResultSetFactory"

        ↓↓↓

    component class="xxx.allcommon.s2dao.S2DaoStatementFactory"
    component class="xxx.allcommon.s2dao.FetchNarrowingResultSetFactory"

      ※注意：jdbc.diconは自動生成対象外です。(dbflute.diconは自動生成)
    _/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/_/


# -------------------------------------------
#                              Task Executing
#                              --------------
6. Execute task of JDBC and Generate

    6-1. jdbc.bat/.shを実行します。./schema以下にproject-schema-xxx.xmlが作成されていたら成功です。
            →中はDBのSchema情報がXML形式で記述されています。

    6-2. generate.bat/.shを実行します。Propertyにて指定した出力DirectoryにSourceが作成されていたら成功です。

    ※実行するJavaのVersionに注意してください。
      例えば、DBFluteが“Java-5.0”でCompileされていて、実行がJDK-1.4の場合はExceptionになります。
      InstallしているJavaのVersionやJAVA_HOMEに指定しているDirectoryなどをご確認下さい。


# -------------------------------------------
#                                Confirmation
#                                ------------
7. Confirm the behavior

    7-1. 確認のために生成されたSourceをCompileしてみて下さい。

    7-2. 確認のためにCompileされたDaoでSelectしてみて下さい。

        ex) BOOKというTableが存在している場合
        ｛PrimaryKeyによる一意検索｝
        /- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
        final BookDao dao = (BookDao)container.getComponent(BookDao.class);
        Book entity = dao.getEntity(new BigDecimal(4));
        - - - - - - /

        ｛ConditionBeanによる一意検索｝
        /- - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
        final BookCB cb = new BookCB();
        cb.query().setBookId_Equal(4);
        final BookDao dao = (BookDao)container.getComponent(BookDao.class);
        Book entity = dao.selectEntity(cb);
        - - - - - - /

# -------------------------------------------
#                                       Other
#                                       -----

たまに、JDBC-TaskにてTable数が膨大な場合に「Memoryが足りない」と怒られることがあります。
(SQLServerにおいて確認されています。OutOfMemoryではなくJDBC-Driverが明示的にthrowしているようです)

その場合は、ANTのHeapSize指定をして下さい。
環境変数'ANT_OPTS'を追加して、'-Xmx256M'を設定します。
※これはDBFluteの機能ではなくANTの機能です。
