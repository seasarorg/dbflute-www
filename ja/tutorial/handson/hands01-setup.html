Eclipseプロジェクトの作成

script配下の maven-eclipse.bat|sh を実行して、コンソールのログを見て正常終了が確認できたら、Eclipse 上でプロジェクトをリフレッシュ(F5)します。すると、その Eclipse プロジェクトが Maven 管理された Java プロジェクトになり、既に存在している ContainerTestCase や PlainTestCase が正常にコンパイルされます。

ソースフォルダの順序調整

ソースフォルダの順序がいまいちな場合は、.classpath を修正して順序を調整しましょう。 (src/main/java, src/main/resources, src/test/java, src/test/resource の順に直す)

デフォルトではドット "."で始まるファイルは見えないようになっているので、パッケージエクスプローラのフィルタの設定で一時的に見えるようにします。 パッケージエクスプローラの右上の下矢印っぽいマークの部分から Filters を選択します。以前に一度で設定をしたことがある場合は、Filters の上に ".* resources" という項目があるかもしれませんので、その場合はそれに付いているチェック外すだけでOKです。 逆に、設定を戻す時はチェックを付けます。

.classpath の修正では、Eclipse のエディタ上のショートカット "alt + 上矢印", "alt + 下矢印" と有効に使うと良いでしょう。現在カーソルのある行が矢印の方向に移動します。

フォーマッタのインポート

Eclipseのデフォルトのフォーマット定義は使いづらいため、tutorial/material/eclipse/formatter-profile.xml を、プロジェクトのプロパティの "Java Code Style" の "Formatter" にてインポート。

ソースコード上では "ctrl + shift + F" でフォーマット。具体的には "Maximum line widths" を 120 に、コメント内のフォーマットを OFF に。

MySQLのセットアップ
