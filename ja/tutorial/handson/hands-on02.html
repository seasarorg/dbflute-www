ハンズオン セクション 01

com.example.dbflute.handson.section.HandsOn02Test クラスを作成してください。 このクラスは ContainerTestCase を継承します。

マスタデータの登録

マスタデータがないことを確認

localdb/login-mysql-root.bat|sh で MySQL にログインして、会員ステータスが一件も存在しないことを確認してください。 

ex) 会員ステータスの件数を取得 @SQL
-- ExampleDBに接続
use exampledb;

-- 会員ステータスのカウント検索
select count(*) from MEMBER_STATUS;

マスタデータを配置して再びReplaceSchema

tutorial/material/dbflute/playsql.zip の data/common/xls/10-master.xls をDBFluteクライアントの playsql 配下に同じディレクトリ構造でコピーし、ReplaceSchemaを実行してください。

テストが通ることを確認

再度テストを実行し、テストが通ってJUnitのバー表示が緑になることを確認してください。 F11 を押すと、さっき実行したテストが再実行されます。これも Quick JUnit のショートカットです。

テストデータの登録

テストデータがあることをアサートするテスト

HandsOn02Test に、会員(MEMBER)のデータをアサートするテストを書いて "落ちること" を確認してください。テストメソッド名は "testテストデータがあること" とします。

アサートに加えて、どんなデータが入っているのか(入るのか)を確認できるように、"会員ID、会員名称、生年月日、会員ステータス名称" をカンマ区切りでログに出力してください。ログの出力は、スーパークラスに log(String msg) というメソッドが定義されているので、それを使ってください。

テストメソッドの雛形

test と打って ctrl + space して enter するとテストメソッドの雛形が作成される。 

ex) テストメソッドのコード補完 @Java

public class HandsOn02Test extends ContainerTestCase {

    test // test と入力して ctrl + space して enter
}
--
public class HandsOn02Test extends ContainerTestCase {

    public void testname() throws Exception { // name の部分が入力モードに
        
    }
}

メソッド名とBehaviorの宣言

テストメソッド名は、"testマスタデータがあること" とし、MemberStatusBhv をインスタンス変数として宣言してください。 Behavior のインスタンスは ContainerTestCase の仕組みでテスト実行前にインジェクションされるようになっています。 

ex) testマスタデータがあること、の実装準備 @Java
public class HandsOn02Test extends ContainerTestCase {

    private MemberStatusBhv memberStatusBhv;

    public void testマスタデータがあること() throws Exception {
        
    }
}

テストコードのお約束コメント

テストの準備(Arrange)、テスト対象の処理の実行(Act)、アサート処理(Assert)がコード上でわかりやすいようにするために、コメントを用意してください。 

ex) 準備、実行、アサートがわかりやすいようなコメントを準備 @Java
public class HandsOn02Test extends ContainerTestCase {

    private MemberStatusBhv memberStatusBhv;

    public void testマスタデータがあること() throws Exception {
        // ## Arrange ##

        // ## Act ##

        // ## Assert ##
    }
}

テストコードの実行

ConditionBeanを使って会員ステータスが存在することをアサートするテストを実装して実行し、テストが "落ちること" を確認してください。テストが正しければ、データはまだ入っていないので例外が発生し、JUnitのバー表示が赤くなるはずです。

テストを実行する際は、そのテストメソッドの中にカーソルがある状態で ctrl + 0 を押すと、そのテストが実行されます。これは Quick JUnit のショートカットです。

テストデータを配置して再びReplaceSchema

tutorial/material/dbflute/playsql.zip の data/ut/xls/20-member.xls, 30-product.xls をDBFluteクライアントの playsql 配下に同じディレクトリ構造でコピーし、ReplaceSchemaを実行してください。

落ちるはずです。20-member.xls を開いてみてみてください。NotNull制約のついている共通カラムのデータがありません。 同じディレクトリに以下の内容で defaultValueMap.dataprop というファイルを作成して、ReplaceSchemaを再実行してみましょう。 (講師による講義あり) 

ex) 共通カラムのデフォルトテストデータの設定 @defaultValueMap.dataprop
map:{
    ; REGISTER_DATETIME = sysdate
    ; REGISTER_USER     = regfoo
    ; REGISTER_PROCESS  = regbar
    ; UPDATE_DATETIME   = sysdate
    ; UPDATE_USER       = upfoo
    ; UPDATE_PROCESS    = upbar
    ; VERSION_NO        = 0
}

テストが通ることを確認

同じ要領で、テストが通ってJUnitのバー表示が緑になることを確認してください。

テストデータの閲覧

せっかくなので、テストデータにさらにアクセスしてみましょう。以下のテストを作成してください。

test会員名称がSで始まる会員を検索
会員名称の昇順で並べる
(検索結果の)会員名称がSで始まっていることをアサート
test会員IDが 1 の会員を検索
一件検索として検索すること
会員IDが 1 であることをアサート
test生年月日がない会員を検索
更新日時の降順で並べる
生年月日がないことをアサート
