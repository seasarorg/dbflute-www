DBFlute-0.6.8 から DBFlute-0.6.9 へ移行する際の注意点を記述します。 


*DBFlute-0.6.9から利用される場合はこのページの説明は無関係です。

# =======================================================================================
#                                                                           DBFlute-0.6.9
#                                                                           =============

今回の移行は、「実際に自動生成してみてコンパイルエラーにならなければOK」となります。
(今回の変更は影響がある場合は必ずコンパイルエラーとして現れます)


1. ClassificationDefinitionのValueLabelListをデフォルトで生成しないようにしました。

Classification定義をTeedaのリストボックスに直接利用できる形の定義がありましたが、
それをオプションにさせて頂きます。
もし、生成したい場合は、build-xxx.propetiesに以下を塚して下さい。

torque.isMakeClassificationValueLabelList = true



2. 自動生成時の数値型のマッピングのデフォルトをBigDecimalからInteger/Longにしました。

jdbcToJavaNativeMapを指定していない場合は、今までは数値は「全てBigDecimal」でした。
これを、TYPEINT/SMALLINT/INTEGERはInteger、BIGINTはLongと変更させて頂きました。

ただ、大分前からですが、テンプレートやEMecha生成のbuild-xxx.propetiesにおいては、
「TYPEINT/SMALLINT/INTEGERはInteger、BIGINTはLong」を明示的に指定していたため、
ほとんどの人は気にする必要がありません。

もし、数値を全てBigDecimalにしていた方は、以下の設定を追加して下さい。
(ずっと昔に設定した方もしくは明示的にInteger/Longの設定を外した方)

torque.jdbcToJavaNativeMap = map:{ ¥
    TYPEINT=java.math.BigDecimal ¥
    ;SMALLINT=java.math.BigDecimal ¥
    ;INTEGER=java.math.BigDecimal ¥
    ;BIGINT=java.math.BigDecimal ¥
}
