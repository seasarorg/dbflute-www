DBFlute-0.4.3 から DBFlute-0.4.4 へ移行する際の注意点を記述します。 


*DBFlute-0.4.4から利用される場合はこのページの説明は無関係です。



# =======================================================================================
#                                                                           DBFlute-0.4.4
#                                                                           =============
# ---------------------------------------------------
#                                        [DBFLUTE-74]
#                                        ------------
以下の条件に“全て”あてはまる方は「ある設定」の追加をお願いします。


A. Java版を利用している。
B. Behaviorを生成している。(恐らくほとんどの人)
C. BehaviorをSetterInjectionで利用している。
D. projectPrefixを利用している。(あまりいないかもしれません)


KeyPointは「projectPrefix」です。

build-xxx.propertiesに「torque.projectPrefix = Ld」と設定すると
全てのClassのClass名の先頭に「Ld」が付加されます。

今まではprojectPrefixを付けても、「dbflute.diconに定義されるBehaviorのComponent名」
にはそのprojectPrefixは付加されませんでした。

なので、LdBookBhvというBehaviorClassが存在している場合、

    protected LdBookBhv bookBhv;
    public void setBookBhv(LdBookBhv bookBhv) {
        this.bookBhv = bookBhv;
    }

という書き方で、SetterInjectionされていました。


DBFlute-0.4.4 では、projectPrefixが設定されていたら、
「dbflute.diconに定義されるBehaviorのComponent名」にもprojectPrefixを付与します。

なので、上記の例は以下のようになります。

    protected LdBookBhv ldBookBhv;
    public void setLdBookBhv(LdBookBhv ldBookBhv) {
        this.ldBookBhv = ldBookBhv;
    }

これは、

[Seasar-user:7068] Re: [DBFlute] 複数ＤＢアクセス時のallcommonについて

による修正です。


無論、今まで「ABCD」の条件に当てはまる方がいる場合に、それまでのProgramを修正するのは大変です。
なので、今まで通りの振る舞いをするPropertyを用意しています。

☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆
torque.isAppendProjectSuffixToComponentName = false
☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆☆

この設定をすることにより、DBFlute-0.4.4以降でも0.4.3以前のと同じように
BehaviorのComponent名にprojectPrefixは付加されません。
(Programを修正する必要はありません)

ex)

# /---------------------------------------------------------------------------
# [PrefixAndSuffix]
#
#   projectPrefix: (NotRequired - Default null)
#     If the property is 'Ld', all class names are 'LdXxx'.
#
#   isAppendProjectSuffixToComponentName: (NotRequired - Default 'true')
#     Default value is true.
#     [true]
#       The component name of bhv and dao have the prefix of project.
#
#     [false]
#       Non.
#
torque.projectPrefix		= Ld
torque.isAppendProjectSuffixToComponentName = false
# ----------------/
