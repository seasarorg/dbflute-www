DBFlute-0.5.6 から DBFlute-0.5.7 へ移行する際の注意点を記述します。 


*DBFlute-0.5.7から利用される場合はこのページの説明は無関係です。



# =======================================================================================
#                                                                           DBFlute-0.5.7
#                                                                           =============

【S2Daoの最新】

対応するS2DaoのDefaultは、1.0.47となります。
それ以前のS2Daoを利用する場合は、

  torque.s2daoVersion = 1.0.46

というように指定して下さい。

  ※都合により、リリース時点(2007/11/10)では、S2Dao-1.0.47はリリースされておりません。
    ですが、S2Dao-1.0.47ではパフォーマンス向上のための修正がされていますので、
    リリース次第S2Dao-1.0.47をご利用されることを強くお奨めします。



【メタデータを利用しない】

S2Dao-1.0.47より導入された「メタデータを利用しない」を利用します。
特に利用上意識する必要はありません。
これにより、初回アクセス時のコストが格段に向上します(UnitTest時に有効)。



【Dao初期化をメソッド単位】

Daoの初期化をメソッド単位にしました。
特に利用上意識する必要はありません。
これにより、初回アクセス時のコストが向上します(UnitTest時に有効)。



【区分値のGreaterXxx/LessXxxを除去】

[DBFLUTE-161]に関して

区分値が設定されている列は、
「GreaterEqual/LessEqual/GreaterThan/LessThan/PrefixSearch/LikeSearch」
の条件設定メソッドを生成しないようにしました。
また、区分値が２つ以下のものは、NotEqualも生成しません。
区分値として利用する列に関しては、ほとんどの場合上記演算子を使うことが無いためです。

torque.isMakeConditionQueryClassificationRestriction = true

で、今まで通りの動きになります。



【非推奨メソッドを整理】

幾つかの非推奨メソッドを整理させて頂きました。
