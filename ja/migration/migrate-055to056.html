DBFlute-0.5.5 から DBFlute-0.5.6 へ移行する際の注意点を記述します。 


*DBFlute-0.5.6から利用される場合はこのページの説明は無関係です。



# =======================================================================================
#                                                                           DBFlute-0.5.6
#                                                                           =============

【S2Daoの最新】

対応するS2DaoのDefaultは、1.0.47となります。
それ以前のS2Daoを利用する場合は、

  torque.s2daoVersion = 1.0.46

というように指定して下さい。

  ※都合により、リリース時点(2007/10/23)では、S2Dao-1.0.47はリリースされておりません。
    ですが、S2Dao-1.0.47ではパフォーマンス向上のための修正がされていますので、
    リリース次第S2Dao-1.0.47をご利用されることを強くお奨めします。


【jdbc.diconのResultSetFactoryとStatementFactory】
今まで、

    component class="org.seasar.extension.jdbc.impl.BasicResultSetFactory"
    component class="org.seasar.extension.jdbc.impl.BasicStatementFactory"

        ↓↓↓

    component class="xxx.allcommon.s2dao.FetchNarrowingResultSetFactory"
    component class="xxx.allcommon.s2dao.S2DaoStatementFactory"

というようにjdbc.diconで設定してもらっていましたが、この設定が不要になりました。
特に変えなくても動作は致しますが、
普通に
    org.seasar.extension.jdbc.impl.BasicResultSetFactory
    org.seasar.extension.jdbc.impl.ConfigurableStatementFactory
を利用してもらって構いません。

これは、FetchNarrowing等の処理を内部に組み込んだためです。
これにより初期設定の手間を一つ省くことができました。
(DBFluteは、jdbc.diconの上記２つのFactoryを利用しないことにご注意下さい)


【BehaviorのTransaction設定】

Behaviorには、Txの付くメソッドにTransactionが自動で掛かるような設定がありますが、
これをDefaultでOFFにさせて頂きました。
理由として：
  ・Diconの解析に時間が掛かる。
  ・その割にはNeedsがほとんど無い。
  ・同じことをApplication側で代替できる。



【CommonColumnの内部構造変更】

以下の設定のDefaultがtrueになります。

# /---------------------------------------------------------------------------
# [Common Column Intercepting On Behavior Filter] (Default 'true')
#   [true]
#     Common column intercepting is executed on behavior filter.
#     And the modifier of behavior's delegateXxx() method is protected!
#
#   [false]
#     Non. (Common column intercepting on interceptor and delegateXxx() is public)
#
# @JavaOnly
torque.isCommonColumnInterceptingOnBehaviorFilter = true
# ----------------/

今後、CommonColumnのFilterは、BehaviorのFilterメソッド内で実行されるようになります。
★====================================================================
★利用者側からは、何も変わる点はありません。あくまで内部動作の話です。
★====================================================================
詳細はこちら



【Entityの子テーブルのListでnullを戻さないように変更】

今まで、Entityのget[ChildTable]List()は、Behaviorでload[ChildTable]List()しない限り、
nullを返していたのですが、その場合は空のListを返すようにしました。
List型なのにnullを戻していたことにより、幾つかの利用者から処理が煩雑であるという声を頂きました。

もし、以前のようにnullが返る仕様でないと都合が悪い場合は
  torque.isOneToManyReturnNullIfNonSelect = true
と指定することにより、以前の動き(nullが帰る)になります。



【Sequence利用時に主キーに値を設定していなくても内部で自動設定するように変更】

torque.isAvailableBehaviorInsertSequenceInjection のデフォルト値をtrueにします。
これにて、
・プログラムで明示的にselectNextVal()で値を取得してEntityの主キーに設定した場合はそれが利用される。
・プログラムでEntityの主キーに値を設定しなかった場合は、Behavior内部で自動で値が設定される。
という動きになります。



【外だしSQLのSQLファイルのエンコーディングをデフォルトでUTF-8に】

torque.daoSqlFileEncoding のデフォルト値をUTF-8にします。
