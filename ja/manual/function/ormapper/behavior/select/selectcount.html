概要

基本概念

ConditionBeanをもとにカウント検索 (select count(*))をします。

ページング条件が設定されていても、ページング条件を無視したカウントを取得します。 また、必ずユニークなカウントを取得します。例えば、PKの存在するテーブルであれば、count(*) の "*" の中には必ず PK カラムが含まれます。PKなしテーブルであれば、"*" には(SpecifyColumnの有無に関わらず)全てのカラムが含まれます。

this - CLOB や TEXT が union 利用できない問題の SpecifyColumn による回避

会話上では、せれくとかうんと と表現します。文脈上、抽象的で伝わりにくい場合は Behaviorのせれくとかうんと(メソッド) や ConditionBeanのせれくとかうんと などと言うとわかりやすいでしょう。

実装方法

実装の流れ

Behaviorの selectCount() を呼び出し、ConditionBeanを指定します。 

ex) selectCount()の実装手順 (Eclipseでコード補完) {MEMBER} @Java
memberBhv.selCo // .selCo と打って enter
--
// セミコロン ';' を打って ctrl(or command) + 2 そして L
memberBhv.selectCount(cb);
--
int count = memberBhv.selectCount(cb);

メソッド仕様

引数

該当のBehaviorに対応するテーブルのConditionBeanとなります。

ConditionBeanが必ず存在することが前提であるため、nullを指定した場合は例外発生します。 検索条件がない検索をする場合でも、必ずConditionBeanのインスタンスが必要です。

設定されている SetupSelect や OrderBy, SpecifyColumn(一部例外あり) など、カウント検索に不要な情報は無視されます。

this - CLOB や TEXT が union 利用できない問題の SpecifyColumn による回避

戻り値

int型となります。検索結果が一件もない場合は 0 が戻ります。

int型(32ビット:2147483648)の精度を超える件数は、サポートされません。

オーバーライド

selectList() と同じような要領となります。

selectList(cb) - オーバーライド

CLOB や TEXT が union 利用できない問題の SpecifyColumn による回避

引数の ConditionBean に SpecifyColumn が指定されている場合、その SpecifyColumn の有無に関わらず、ユニークなカウントが取得されます。よって、基本的には SpecifyColumn は無視される考えてもいいでしょう。

ただ、UnionQuery (union all は除く)の利用でTEXT型やCLOB型のカラムが文法的に許されない DBMS の場合のために、SpecifyColumn でカラムを絞り込むことで回避することが可能です。PKが存在するテーブルにおいてのみ、UnionQuery の select 句では SpecifyColumn で指定されているカラムだけが列挙されます。PK カラムはそもそも SpecifyColumn の暗黙の必須カラムなのでカウント結果は変化しません。

PKなしテーブルでは、この回避技は制限として利用できません。また、ビューも "PKなしテーブル" に含まれますが、additionalPrimaryKeyMap.dfprop などでユニークなカラムに(疑似) PK を設定することでこの回避技を利用できます。
