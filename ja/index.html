DBFluteとは？

  • DBFluteは、S2Daoの利用をサポートするToolです。

    • ※Apache TorqueのGenerator機能をS2Daoに適応させたものです。

DBFluteの機能

  • S2Daoで利用するDao/EntityをDatabaseのSchema情報を利用して自動生成します。
  • S2Daoの“DTOによるSQL自動生成”の機能を拡張したDTOを自動生成します。
  • DatabaseのSchema情報を表示する'SchemaHTML'を自動生成します。Sample(LDB)
  • S2Daoの“外だしSQL”(SQLファイル)から対応するEntityを自動生成します。Tips: Sql2Entity

DBFluteのPolicy

    【O/R-MapperとしてPolicy】
    DBFluteは、SQL文を自動生成する機能を有しますが、SQL文を隠蔽するつもりはありません。

    【実装Policy 】
    [簡単なSQL＋複雑だけれども定型的なSQL]
    　　ConditionBeanでTypeSafe実装
    [複雑なSQL]
    　　Sql2Entityの支援と共に“外だしSQL”で実装

Support Database

  • [自動生成機能がサポートされるDatabase]
      • axion, cloudscape, db2, db2400, hypersonic, interbase, mssql
      , mysql, oracle, postgresql, sapdb, sybase, firebird, derby, h2

      ※設定Fileに定義する際のDB名で記述しています。
      ※“mssql”はSQLServerのことです。
  • [ConditionBeanのPaging検索がSQLレベルでサポートされるDatabase]

    • “何件から何件まで”検索
    • Oracle : rownumを利用
    • Firebird : first構文を利用
    • MySQL : limit構文を利用
    • PostgreSQL : offset-limit構文を利用
    • H2 : offset-limit構文を利用

    “何件まで”検索
    • SQLServer : top構文を利用
    • DB2 : fetch first x rows only構文を利用

    ※SQLレベルのサポートがされていないDatabaseに関しては、
    　S2Pagerと同様の方式でPagingを実現します。
