DBFluteとは？

  • DBFluteは、S2Daoの利用をサポートするToolです。

    • ※Apache TorqueのGenerator機能をS2Daoに適応させたものです。

DBFluteの機能

  • S2Daoで利用するDao/EntityをDatabaseのSchema情報を利用して自動生成します。
  • S2Daoの“DTOによるSQL自動生成”の機能を拡張したDTOを自動生成します。
  • DatabaseのSchema情報を表示する'SchemaHTML'を自動生成します。Sample(LDB)
  • S2Daoの“外だしSQL”(SQLファイル)から対応するEntityを自動生成します。Tips: Sql2Entity

DBFluteのPolicy

    DBFluteは、S2Daoの利用を促進することを最大の目的とします。

    DBFluteは、SQL文を自動生成する機能を有しますが、SQL文を隠蔽するつもりはありません。 簡単なSQLは自動生成し、複雑なものはS2Daoの“外だしSQL”(外部FileにSQLを記述するやり方) を利用して思う存分SQLを書くべきだと考えています。

    　　注意点：
    　　　[DBFLUTE-51]の「j2eeDiconResourceのDefault値をj2ee.diconに戻す。」
    　　　対象：j2eeDiconResourceをbuild.propertiesに指定せずにデフォルト値の「jdbc.dicon」を利用されている方
    　　　作者が勘違いして、jdbc.diconをデフォルト値にとるようにしてしまいましたが、
    　　　S2の標準では Seasarのjarの内部に j2ee.dicon が存在していて、それが jdbc.dicon をincludeしているため、
    　　　基本、dbflute.diconからは j2ee.dicon をincludeすれば独自で作成したjdbc.diconが読み込まれることになります。
    　　　S2の標準に従うために、真に申し訳御座いませんが、デフォルト値を「j2ee.dicon」に戻させて頂きます。
    　　　　→ 参考 - [Seasar-user:6168] DBFluteの０．４．０以降の設定について
    　　　　→ 参考 - こちら
    　　
    　　　既にデフォルト値のjdbc.diconを利用している方：
    　　　　＜S2Containerのjar内部に存在するj2ee.diconを利用できる方＞
    　　　　　 jdbc.diconに定義されている「S2Containerのjar内部に存在するj2ee.diconと重複するComponent」の定義を
    　　　　　 削除して下さい。そうすると、dbflute.dicon → j2ee.dicon → jdbc.dicon という参照関係が成立します。
    　　　　＜S2Containerのjar内部に存在するj2ee.diconを利用しない方＞
    　　　　　 お手数ですが、build.propertiesにj2eeDiconResourceNameプロパティで明示的に「jdbc.dicon」とするか、
    　　　　　 既存の「jdbc.dicon」を「j2ee.dicon」に名前を変更するようにお願いします。
    　　
    　　　ご迷惑お掛けしました。

Support Database

  • [自動生成機能がサポートされるDatabase]
      • axion, cloudscape, db2, db2400, hypersonic, interbase, mssql
      , mysql, oracle, postgresql, sapdb, sybase, firebird, derby

      ※設定Fileに定義する際のDB名で記述しています。
      ※“mssql”はSQLServerのことです。
  • [ConditionBeanのPaging検索がSQLレベルでサポートされるDatabase]

    • “何件から何件まで”検索
    • Oracle : rownumを利用
    • Firebird : first構文を利用
    • MySQL : limit構文を利用
    • PostgreSQL : offset-limit構文を利用

    “何件まで”検索
    • SQLServer : top構文を利用
    • DB2 : fetch first x rows only構文を利用

    ※SQLレベルのサポートがされていないDatabaseに関しては、
    　S2Pagerと同様の方式でPagingを実現します。
