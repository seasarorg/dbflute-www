DBFluteとは？

  • DBFluteは、S2Daoの利用をサポートするToolです。

    • ※Apache TorqueのGenerator機能をS2Daoに適応させたものです。

DBFluteの機能

  • S2Daoで利用するDao/EntityをDatabaseのSchema情報を利用して自動生成します。
  • S2Daoの“DTOによるSQL自動生成”の機能を拡張したDTOを自動生成します。

DBFluteのPolicy

    DBFluteは、S2Daoの利用を促進することを最大の目的とします。

    DBFluteは、SQL文を自動生成する機能を有しますが、SQL文を隠蔽するつもりはありません。 簡単なSQLは自動生成し、複雑なものはS2Daoの“外だしSQL”(外部FileにSQLを記述するやり方) を利用して思う存分SQLを書くべきだと考えています。

お知らせ

  • 【2006-10-19】 DBFlute-0.1.05では、ConditionBeanの条件をInlineViewとして設定できるようにします。
    • 今までは、結合先の列のWhere句条件は必ずBaseTableの件数を絞られていました。そうでなければならないこともありますが、
    NullableであるFK列での参照など、結合先Tableのみが単純に絞られてBaseTableの件数は絞りたくない場合があります。
    その時は、結合先Tableはfrom句のInlineViewにて絞らなければなりません。(結合先の論理削除を絞る場合など)
    ConditionBeanではこのように指定できます。
    　　// 従来：条件は通常のWhere句に付加される。BaseTableもこの条件で絞られる。
    　　cb.query().queryForeignTable().setActiveYn_Equal("y");
    　　 // 追加：条件はInlineViewのWhere句に付加される。BaseTableはこの条件で絞られない。
    　　cb.query().queryForeignTable().setActiveYn_Equal_AsInline("y");

  • 【2006-10-19】 DBFlute-0.1.05では、Classificationの便利MethodをConditionBeanにも追加しました。
    • 今までは、Entityには設定Methodや判定Methodなどの便利Methodが作成されていましたが、
    ConditionBeanに区分値の条件付加Methodを作成するようにしました。
    ConditionBeanではこのように指定できます。
    　　cb.query().setActiveYn_Equal_Yes();
    　　cb.query().setActiveYn_Equal_No();

  • 【2006-10-16】 DBFlute-0.1.04をReleaseしました。内容は以下の通りです。
    • {Java/C#}
    　A. JDBCよりAutoIncrement情報を取得してID-Annotationを自動付加するように修正。｛Tips: Identity｝
    　B. Classificationの定義で、実際のTableに登録されているDataを取得できるように修正。｛Tips: Classification｝

  • 【2006-10-12】 DBFlute-0.1.04では、JDBCよりAutoIncrement情報を取得してID-Annotationを自動付加するようにします。
    • 今までは、build.propertiesにIdentityの定義{Table=Column;...}を記述しなければならず、Tableが増えたり消えたりする度に
    build.propertiesの書き換え作業が発生していたのですが、今後はその必要がなくなります。
    但し、JDBCの実装によっては正確に取得できない場合もあるかも知れないので、Identityの定義は残しておきます。
    JDBCのAutoIncrement情報に無くても、build.propertiesのIdentityの定義で指定されている場合はID-Annotationを付与します。

Support Database

  • [自動生成機能がサポートされるDatabase]
      • axion, cloudscape, db2, db2400, hypersonic, interbase, mssql
      , mysql, oracle, postgresql, sapdb, sybase, firebird, derby

      ※設定Fileに定義する際のDB名で記述しています。
      ※“mssql”はSQLServerのことです。
  • [ConditionBeanのPaging検索がSQLレベルでサポートされるDatabase]


    • “何件から何件まで”検索
    • Oracle : rownumを利用
    • Firebird : first構文を利用
    • MySQL : offset構文を利用
    • PostgreSQL : offset-limit構文を利用

    “何件まで”検索
    • SQLServer : top構文を利用
    • DB2 : fetch first x rows only構文を利用

    ※SQLレベルのサポートがされていないDatabaseに関しては、
    　S2Pagerと同様の方式でPagingを実現します。
