前提条件

前提となる環境

Javaアプリケーションが実行できること
DBFluteは Apache Ant を利用するため、Java-5.0 以上の JDK か JRE の環境が必要です。
アプリケーションのEclipseプロジェクト(Javaプロジェクト)が存在すること
サポートされているDIコンテナのいずれかの動作環境が整っている必要があります。
データベース上に(空の)スキーマが存在すること
ReplaceSchemaを利用する場合は、スキーマ上にはテーブルは全くなしで(空のスキーマ)、 テーブルをCreateするDDL文が準備(ERDツールで生成されているなど)されていればOKです。

複数DB対応について

複数DB対応をする場合、事前にポイントを抑えてからセットアップをするようにして下さい。

複数DBの利用

セットアップ手順

DBFluteクライアントの作成(+ モジュール配置)

DBFluteモジュールの配置

最新バージョンのDBFluteモジュールをダウンロードし、ZIPを解凍してプロジェクト直下の ./mydbflute/dbflute-x.x.x ディレクトリ配下に配置して下さい。

DBFluteの最新バージョン
ex) foo-logicプロジェクトにDBFluteモジュール(1.0.0)を配置
foo-logic
 |-src/main/java
 |-src/main/resources
 |-Referenced Libraries
 |-...
 |-mydbflute
    |-dbflute-1.0.0
       |-ant
       |-...
       |-templates
       |-build-torque.xml
       |-product-is-dbflute-1.0.0

DBFluteクライアントの配置

配置したDBFluteモジュールの etc/client-template/dbflute_dfclient.zip を解凍してプロジェクト直下に配置して下さい。 

ex) foo-logicプロジェクトにDBFluteクライアントを作成
foo-logic
 |-src/main/java
 |-...
 |-dbflute_dfclient
    |-dfprop
    |-...
 |-mydbflute
    |-dbflute-1.0.0

クライアントプロジェクト名の設定

以下の箇所を任意のクライアントプロジェクト名に変更して下さい(全て同名の必要があります)。

  • dbflute_dfclientディレクトリ名の dfclient 部分
  • dbflute_dfclient/_project.bat(sh) の MY_PROJECT_NAME の値
  • dbflute_dfclient/build.properties の torque.project の値

クライアントプロジェクト名は対象とするデータベースを識別できる名前が好ましいです。もし思いつかないでのあればデフォルトの dfclient のままでも構いません。 

ex) クライアントプロジェクト名を foodb に
foo-logic
 |-src/main/java
 |-...
 |-dbflute_foodb       // ディレクトリ名を dbflute_foodb に
    |-dfprop
    |-...
    |-_project.bat     // MY_PROJECT_NAME の値をfoodbに
    |-_project.sh      // MY_PROJECT_NAME の値をfoodbに
    |-build.properties // torque.project の値をfoodbに
    |-...
 |-mydbflute
    |-dbflute-1.0.0

基本プロパティの設定

DBFluteプロパティ basicInfoMap.dfprop を参考に以下の項目を設定して下さい。

basicInfoMap.dfprop
database
データベースの種類(MySQLやPostgreSQLなど)
targetContainer
DIコンテナの種類(SeasarやSpringなど)
packageBase
自動生成されるクラスの基底パッケージ

データベース接続プロパティの設定

DBFluteプロパティ databaseInfoMap.dfprop を参考に以下の項目を設定して下さい。

databaseInfoMap.dfprop
driver
DBFluteが利用するJDBCドライバのFQCN
url
自動生成対象データベースの接続URL
schema
接続スキーマ
user
接続ユーザ
password
接続ユーザのパスワード
DBMSごとの取扱い

その後の設定

ひとまず、アプリケーション上でDBFluteのクラスを利用することができるようになりました。 しかし、ディベロッパーに横展開するまでには、まだあともうちょい頑張りが必要です。 より良い環境で実装してもらうために、現場フィットな機能を整えましょう！
