複数DB対応のポイント

複数DB対応時のポイントを説明します。

既に出来上がってる複数DB対応プロジェクトを参考にすることもできます(以下からチェックアウトが可能)。

  • SVNリポジトリ：https://www.seasar.org/svn/sandbox/dbflute/
  • 参考プロジェクト：dbflute-multipledb-example

複数DBの前提

ここでいう「複数DB」とは、「テーブル構造の違う別のデータベース(スキーマ)の固定的な複数DB」を示します。(同じテーブル構造のDB２つで状況によってInsert先を変えるような動的な複数DBではありません)

DBFluteでの実現方法

非常に単純です。DBFluteクライアントを二つ作成します。

以下は一つのプロジェクトに二つのDBFluteクライアントを作成した場合の構造例です： 


   ex) Javaの場合：
     xxxproject
         +- .settings
         +- dbflute_[project1]        // DBFluteクライアント1
         |     +- dfprop
         |     +- log
         |     +- ...
         +- dbflute_[project2]        // DBFluteクライアント2
         |     +- dfprop
         |     +- log
         |     +- ...
         +- mydbflute                 // DBFluteモジュールの格納場所
         |     +- dbflute-0.6.5
         +- src/main/java             // ソースディレクトリ

   ex) C#の場合：
     xxxproject
         +- dbflute_[project1]        // DBFluteクライアント1
         |     +- dfprop
         |     +- log
         |     +- ...
         +- dbflute_[project2]        // DBFluteクライアント2
         |     +- dfprop
         |     +- log
         |     +- ...
         +- lib
         +- mydbflute                 // DBFluteモジュールの格納場所
         |     +- dbflute-0.6.5
         +- source/xxx.sln            // ソースディレクトリ

クラスやリソースファイルの識別

二つのDBFluteクライアントでリソースがバッティングしないように、以下のDBFluteプロパティを考慮して下さい。

  • (basicInfoMap)projectPrefix = [prefix of class name] --> Default: null
  • (dependencyInjectionMap)dbfluteDiconFileName = [name] --> Default: dbflute.dicon *Javaのみ
  • (dependencyInjectionMap)j2eeDiconResourceName = [name] --> Default: j2ee.dicon *Javaのみ
  • (dependencyInjectionMap)quillDataSourceName = [data-source name] --> Default: null(なし) *C#のみ(0.8.6より実装)

projectPrefix

自動生成されたクラスのクラス名の先頭になんらかの文字を付けたい場合に指定します。デフォルトは「null(なし)」です。複数DB対応時の２つ目以降のDBの場合において識別しやすくするために設定することが多いです。

dbfluteDiconFileName

dbflute.diconの名前を設定します。デフォルトは「dbflute.dicon」です。複数DB対応時の２つ目以降のDBの場合において「dbflute-[dbname].dicon」とすることが多いです。

j2eeDiconResourceName

利用するj2ee.diconを設定します。デフォルトは「j2ee.dicon」です。複数DB対応時の２つ目以降のDBの場合において「j2ee-[dbname].dicon」とすることが多いです。

quillDataSourceName

Quillの設定で定義したデータベース(スキーマ)のデータソース名です。デフォルトは「null(なし)」です。C#限定のプロパティです。単一DBの場合は設定する必要性はありません(してもOK)。複数DB対応時は必ず設定する必要があります。

設定の適用方式について

一つ目のDBFluteクライアントでは上記のプロパティは設定せずに二つ目のDBFluteクライアントで設定するやり方と、両方のDBFluteクライアントで設定して対比的にするやり方と二つあります。これは、アプリケーションからみたときの二つのDBの重要度の比重によるかもしれません。二つ目のDBがテーブルが２、３個の明らかに再サイド的なDBであれば前者が良いかもしれませんが、二つとも同じくらい重要なDBで恒久的なのであれば後者が良いかもしれません。

外だしSQLのパッケージ指定

一つのプロジェクトで二つ以上のDBFluteクライアントを扱うときは、外だしSQLのパッケージを明確に指定する必要があります。 (もう一方のDBのSQLファイルを読み込んでしまうため)

「outsideSqlDefinitionMap.dfprop」にてsqlPackageを指定します。これを双方のDBFluteクライアントに定義して下さい。 

ex) 外だしSQLのパッケージをpackageBaseに絞るmap:{
    ; sqlPackage = $$PACKAGE_BASE$$
}

それぞれのDBごとにプロジェクトを構築する場合はこの設定は不要です。

Diconの複数DB設定(Java)

申し訳ございません。ドキュメント作成中です。dbflute-multipledb-exampleを参考にして下さい。

Quillの複数DB設定(C#)

申し訳ございません。ドキュメント作成中です。
