DBFluteにおけるOracleの扱い

概要

DBFluteを利用する上でのOracleの扱いを述べます。

※OracleのバージョンはOracle-9i以上を前提として記載しています。

JDBCタスクで利用するJDBCドライバ

JDBCドライバは「ojdbc14.jar」を利用しています。

JDBCタスクの接続設定(./dfprop/databaseInfoMap.dfprop)

OracleにつなぐJDBCタスクの接続設定は以下の通りです。スキーマ名は必ず大文字で指定して下さい。 

ex) Oracleに接続{port=1521} @databaseInfoMap.dfprop
map:{
    ; driver          = oracle.jdbc.driver.OracleDriver
    ; url             = jdbc:oracle:thin:@localhost:1521:[SID]
    ; schema          = [schema]
    ; user            = [dbuser]
    ; password        = [dbpassword]
}

主キーで連番の利用

Oracleは連番生成の仕組みとしてSequenceを利用します。

DBFluteでのSequence設定

設定ファイルにテーブルとシーケンスの関連づけを行う必要があります。./dfprop/sequenceDefinitionMap.dfpropに 

ex) Sequenceの設定 @sequenceDefinitionMap.dfprop
map:{
    ; PURCHASE     = SEQ_PURCHASE
    ; MEMBER       = SEQ_MEMBER
    ; MEMBER_LOGIN = SEQ_MEMBER_LOGIN
    ; PRODUCT      = SEQ_PRODUCT
}

DBFluteでのInsert時のSequence利用

Entityの主キーのプロパティに値を何も設定せずに実行すると内部的にSequenceで連番を取得して設定されます。実行後に発行された連番をそのEntityの主キーのプロパティから取得できます。 

ex) Sequenceを利用してInsert {MEMBERをInsert} @Java
Member member = new Member();
member.setMemberName("Billy Joel");
member.set... // その他主キー以外にプロパティに値を設定
memberBhv.insert(member);
Integer memberId = member.getMemberId();// 発行された連番の値を取得

もちろん、Sequenceを明示的に取得してInsert前のEntityに設定することも可能です。Insert前にInsert予定のPKを使って何か処理をする場合に有効です(Identityではできないことです)。 

ex) Sequenceを利用してInsert {MEMBERをInsert} @Java
Member member = new Member();
member.setMemberId(memberBhv.selectNextVal());// Sequenceを取得して主キーに設定
member.setMemberName("Billy Joel");
member.set... // その他主キー以外にプロパティに値を設定
memberBhv.insert(member);

補足：Create Sequence文でのSequence作成

Create Sequence文では以下のように指定します。 

ex) OracleでCreate Sequence文でSequenceを作成 @DDL
CREATE SEQUENCE SEQ_PURCHASE START WITH 1000 INCREMENT BY 1;
CREATE SEQUENCE SEQ_MEMBER START WITH 1000 INCREMENT BY 1;
CREATE SEQUENCE SEQ_MEMBER_LOGIN START WITH 1000 INCREMENT BY 1;
CREATE SEQUENCE SEQ_PRODUCT START WITH 1000 INCREMENT BY 1;

ページングの実現方法

rownumを利用しています。 

ex) ConditionBeanでページング @SQL
select *
  from (
select base.*, rownum as rn
  from (

    select ... from ... where ... order by ..

       ) base
       )
 where rn > 80
   and rn <= 100;

OutsideSqlのManualPaging(手動ページング)を利用する場合は以下のように実装します。 

ex) OutsideSqlのManualPaging(手動ページング)でページング @SQL
select *
  from (
select base.*, rownum as rn
  from (

    select ... from ... where ... order by ..

       ) base
       )
 where rn > /*$pmb.pageStartIndex*/80
   and rn <= /*$pmb.pageEndIndex*/100;

OracleのCLOBを使う場合の注意点

OracleのCLOBが含まれているテーブルを検索する場合は、以下の設定をする必要があります。

  1. CLOB用のValueTypeコンポーネントをjdbc.diconに設定する
  2. 該当テーブルのExEntityに設定したCLOB用のValueTypeのValueTypeアノテーションを付ける
ex) CLOB用のValueTypeコンポーネントをjdbc.diconに設定する @jdbc.dicon
<component name="stringClobType" class="org.seasar.extension.jdbc.types.StringClobType" />
	
ex) 該当テーブルのExEntityに設定したCLOB用のValueTypeのValueTypeアノテーションを付ける @ExEntity
/** 退会理由テーブルの「退会理由テキスト」プロパティにstringClobTypeを設定 */
public static String withdrawalReasonText_VALUE_TYPE = "stringClobType";

このようにすると、OracleのCLOBがString型で取得できるようになります。

2008年1月現在、上記の設定をDBFlute内部で自動で組み込むことを検討しています。そうすれば、上記のことをユーザの方々が意識をする必要がなくなります。その場合は、CLOB用のValueTypeコンポーネントはdbflute.diconに設定され、ValueTypeアノテーションはBsEntityに設定しようかと考えています。
