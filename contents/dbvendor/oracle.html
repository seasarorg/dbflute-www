DBFluteにおけるOracleの扱い

概要

DBFluteを利用する上でのOracleの扱いを述べます。

※バージョンはOracle-9i以上を前提として記載しています。

JDBCタスクで利用するJDBCドライバ

JDBCドライバは「ojdbc5.jar」を利用します。

JDBCドライバは同梱されていませんので、JDBCドライバのオフィシャルサイトからダウンロードして、 DBFluteクライアントの「extlib」配下に配置して下さい。(extlibディレクトリは手動で作成)

JDBCタスクの接続設定(./dfprop/databaseInfoMap.dfprop)

JDBCタスクの接続設定は以下の通りです。スキーマ名は必ず大文字で指定して下さい。 

ex) 接続{port=1521} @databaseInfoMap.dfprop
map:{
    ; driver          = oracle.jdbc.OracleDriver
    ; url             = jdbc:oracle:thin:@localhost:1521:[SID]
    ; schema          = [schema-as-UpperCase]
    ; user            = [dbuser]
    ; password        = [dbpassword]
}
	※[xxx]部分にプロジェクト独自の値を設定。[]で囲う必要はない。

主キーで連番の利用

Oracleは連番生成の仕組みとしてSequenceを利用します。

DBFluteでのSequence設定
詳しくはDBFluteのSequenceをご覧下さい。

補足：OracleでSequenceを作成

Create Sequence文では以下のように指定します。(オプションは割愛) 

ex) OracleでSequenceを作成 @DDL
CREATE SEQUENCE SEQ_MEMBER START WITH 1 INCREMENT BY 1;

補足：OracleでSequenceを取得

Create Sequence文では以下のように指定します。 

ex) OracleでSequenceを取得 @SQL
select SEQ_MEMBER.nextval from dual;

ページングの実現方法

rownumを利用しています。 

ex) ConditionBeanでページング @SQL
select *
  from (
select base.*, rownum as rn
  from (

    select ... from ... where ... order by ...

       ) base
       )
 where rn > 80
   and rn <= 100;

OutsideSqlのManualPaging(手動ページング)を利用する場合は以下のように実装します。 

ex) OutsideSqlのManualPaging(手動ページング)でページング @SQL
select *
  from (
select base.*, rownum as rn
  from (

    select ... from ... where ... order by ...

       ) base
       )
 where rn > /*$pmb.pageStartIndex*/80
   and rn <= /*$pmb.pageEndIndex*/100;

OracleのCLOBを使う場合の注意点

以下内容ですが、DBFlute-0.6.2よりDBFluteが内部的に解決するようにしたため、利用者が意識する必要はなくなりました。

OracleのCLOBが含まれているテーブルを検索する場合は、以下の設定をする必要があります。(DBFlute-0.6.1まで)

  1. CLOB用のValueTypeコンポーネントをjdbc.diconに設定する
  2. 該当テーブルのExEntityに設定したCLOB用のValueTypeのValueTypeアノテーションを付ける
ex) CLOB用のValueTypeコンポーネントをjdbc.diconに設定する @jdbc.dicon
<component name="stringClobType" class="org.seasar.extension.jdbc.types.StringClobType" />
	
ex) 該当テーブルのExEntityに設定したCLOB用のValueTypeのValueTypeアノテーションを付ける @ExEntity
/** 退会理由テーブルの「退会理由テキスト」プロパティにstringClobTypeを設定 */
public static String withdrawalReasonText_VALUE_TYPE = "stringClobType";

このようにすると、OracleのCLOBがString型で取得できるようになります。

2008年1月現在、上記の設定をDBFlute内部で自動で組み込むことを検討しています。そうすれば、上記のことをユーザの方々が意識をする必要がなくなります。その場合は、CLOB用のValueTypeコンポーネントはdbflute.diconに設定され、ValueTypeアノテーションはBsEntityに設定しようかと考えています。

バッチ更新の排他制御の注意点

以下内容ですが、DBFlute0.7.9よりStatement.getUpdateCount()を利用することにより排他制御ができるようになっております。

BehaviorのbatchUpdate()ですが、Oracleの場合は排他制御は効きません。

これは、OracleのJDBCドライバがバッチ更新の結果の値を正確に戻さないためです。DBFluteとしてはどうにもできないため、どうしても排他制御の判定をすることができません。(もし、正常に戻すJDBCドライバがありましたらご一報頂けるとありがたいです)
