DBFluteにおけるSQLServerLの扱い

概要

DBFluteを利用する上でのSQLServerの扱いを述べます。

※バージョンはSQLServer2000以上を前提として記載しています。

JDBCタスクで利用するJDBCドライバ

JDBCドライバは「sqljdbc.jar(or sqljdbc4.jar)」を利用します。

JDBCドライバは同梱されていませんので、JDBCドライバのオフィシャルサイトからダウンロードして、 DBFluteクライアントの「extlib」配下に配置して下さい。(extlibディレクトリは手動で作成)

DBFluteモジュール自体が動作するJREが「5.0」の場合は「sqljdbc.jar」を、 「6.0」の場合は「sqljdbc4.jar」を使う必要がありますのでご注意下さい(同梱していない理由の一つ)。

JDBCタスクの接続設定(./dfprop/databaseInfoMap.dfprop)

JDBCタスクの接続設定は以下の通りです。 

ex) 接続{port=3306} @databaseInfoMap.dfprop
map:{
    ; driver          = com.microsoft.sqlserver.jdbc.SQLServerDriver
    ; url             = jdbc:sqlserver://localhost:1433;DatabaseName=[dbname];
    ; schema          = dbo
    ; user            = [dbuser]
    ; password        = [dbpassword]
}
	※[xxx]部分にプロジェクト独自の値を設定。[]で囲う必要はない。

主キーで連番の利用

連番生成の仕組みとしてIdentityを利用します。

DBFluteでのIdentity設定
詳しくはDBFluteのIdentityをご覧下さい。

実装制限

ConditionBeanで「Unionを使ったPaging」を実行することができません。

Example

SQLServerを利用したDBFluteのExampleプロジェクト(Java)が存在します。ぜひそちらを参考にしてみて下さい。
