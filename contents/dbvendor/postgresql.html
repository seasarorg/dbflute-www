DBFluteにおけるPostgreSQLの扱い

概要

DBFluteを利用する上でのPostgreSQLの扱いを述べます。

※バージョンはPostgreSQL-8.1以上を前提として記載しています。

JDBCタスクで利用するJDBCドライバ

JDBCドライバはDBFlute0.6.6までは「postgresql-8.1-407.jdbc3.jar」で、DBFlute-0.6.7からは「postgresql-8.3-603.jdbc3.jar」を利用しています。

JDBCタスクの接続設定(./dfprop/databaseInfoMap.dfprop)

JDBCタスクの接続設定は以下の通りです。 

ex) 接続{port=5432} @databaseInfoMap.dfprop
map:{
    ; driver          = org.postgresql.Driver
    ; url             = jdbc:postgresql://localhost:5432/[dbname]
    ; schema          = 
    ; user            = [dbuser]
    ; password        = [dbpassword]
}

主キーで連番の利用

PostgreSQLは連番生成の仕組みとしてSequenceを利用します。

DBFluteでのSequence設定
後述の「PostgreSQLでのSequenceについて」をご覧下さい。

PostgreSQLでのSequenceについて

詳しくはDBFluteのSequenceをご覧下さい。但し、以下に補足があります。

通常Sequenceは明示的にSequenceとテーブルの関連付けを設定ファイルに記述する必要があるのですが、 Serial型を利用する場合はDBFluteが特別な処理でその関連付けを自動判別しているのでその設定は不要です。

ページングの実現方法

limit/offset句を利用しています。
