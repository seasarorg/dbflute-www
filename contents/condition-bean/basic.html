ConditionBeanの基本

ConditionBeanとは？

そもそもConditionBeanとは？に関してはこちら

基本的な使い方

  • A. ConditionBeanを生成
    • そのSQLの基点テーブルは何か？
  • B. 条件の設定
    • 関連テーブルは何を取得したいのか？(Select句)
    • どのような絞り込みをしたいのか？(Where句 ※一部From句)
    • どのような並べ替えをしたいのか？(OrderBy句)
    • ページングや更新ロックの取得などしたいのか？
  • C. Behaviorのメソッドを呼ぶ
    • 一覧検索なのか一件検索なのか？

ex) 基点テーブルを会員(MEMBER)とした場合 @Java
// = = = = = = = = = = = = =
// A. ConditionBeanを生成
// = = = = = = = = = = = = =
MemberCB cb = new MemberCB(); // 基点テーブルは会員

// = = = = = = = = = = = = =
// B. 条件の設定
// = = = = = = = = = = = = =
cb.setupSelect_MemberStatus(); // 会員ステータスを結合して取得(Select句に設定)
cb.query().setMemberName_PrefixSearch("S"); // 会員名が'S'で始まること
cb.query().addOrderBy_MemberBirthday_Desc(); // 会員の生年月日の降順で並べる

// = = = = = = = = = = = = =
// C. Behaviorのメソッドを呼ぶ
// = = = = = = = = = = = = =
List<Member> memberList = memberBhv.selectList(cb); // 一覧検索

for (Member member : memberList) {
    ... = member.getMemberName();
    ...
}

SQL実装の思考の流れ

ここで紹介されている実装の上での思考の流れは、これはConditionBeanに限らず通常SQLを書く際にそのまま通じる大事な要素です。逆に解釈すると、ConditionBeanは「SQLでの思考の流れ」をそのままタイプセーフな実装の世界に展開し、「開発者の思考の流れ」の安全なレールを敷いていると言えます。

具体的な使い方

SetupSelect(結合先取得)

これは先ほどの「基本的な使い方」の「関連テーブルは何を取得したいのか？(Select句)」に相当する機能です。詳しくはこちら

ConditionQuery(条件設定)

これは先ほどの「基本的な使い方」の「どのような絞り込みをしたいのか？(Where句 ※一部From句)」や「どのような並べ替えをしたいのか？(OrderBy句)」に相当する機能です。詳しくはこちら
