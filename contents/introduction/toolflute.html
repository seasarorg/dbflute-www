ツールとしてのDBFlute

構造

DBFluteを利用する際は、二つのディレクトリが存在します。

DBFluteクライアント
「dbflute_xxx」ディレクトリ
DBFluteモジュール
「mydbflute/dbflute-x.x.x」ディレクトリ
ex) Javaの場合：     xxxproject
         +- .settings
         +- dbflute_[project]         // DBFluteクライアント
         |     +- dfprop
         |     +- log
         |     +- ...
         +- mydbflute
         |     +- dbflute-0.8.8       // DBFluteモジュール
         +- src/main/java             // ソースディレクトリ	
ex) C#の場合：     xxxproject
         +- dbflute_[project]         // DBFluteクライアント
         |     +- dfprop
         |     +- log
         |     +- ...
         +- lib
         +- mydbflute
         |     +- dbflute-0.8.8       // DBFluteモジュール
         +- source/xxx.sln            // ソースディレクトリ

DBFluteモジュールは設置したら基本的に意識する必要はありません。プロジェクト独自の設定と実行環境は全てDBFluteクライアントに含まれます。

DBFluteクライアントがDBFluteモジュールを利用(参照)します。どのバージョンのDBFluteモジュールを利用するかは、 DBFluteクライアント配下の「_project.bat(.sh)」にて「DBFLUTE_HOME」という環境変数で指定されます。

タスク一覧

DBFluteは、内部でApache Antを利用して、それぞれのタスクを実行します。

  • JDBC : 自動生成のためのDBスキーマ情報の取得
  • Generate : DBスキーマ情報からのクラスの自動生成
  • Sql2Entity : 外だしSQLからのクラスの自動生成
  • ReplaceSchema : DBスキーマを初期化してDDL文を流しテストデータまで一気に登録
  • Doc : DBスキーマ情報をHTMLのテーブル一覧として出力 *サンプル
  • OutsideSqlTest : 外だしSQLを2WaySQLとしてそのまま実行し、SQLの妥当性をチェック

実行の仕方

DBFluteクライアント配下にそれぞれWindows用、Unix用(MacOSXも含む)に「.bat」と「.sh」が用意されています。

  • JDBC : jdbc.bat(.sh)
  • Generate : generate.bat(.sh)
  • Sql2Entity : sql2entity.bat(.sh)
  • ReplaceSchema : replace-schema.bat(.sh)
  • Doc : doc.bat(.sh)
  • OutsideSqlTest : outside-sql-test.bat(.sh)

実行すると、コンソールにログが表示されます。詳細を追う必要はないですが、実行が正常終了したのかどうかは 必ず確認するようにして下さい。もし、エラーが発生した際にログの詳細をみて状況を把握します。 DBFluteクライアントの「./log/dbflute.log」にも同じログが出力されていますので、 コンソールが見づらい環境の場合(特にコマンドプロンプト)は、そちらをテキストエディタで開いて見ると良いでしょう。

Windows上のEclipseであれば、Eclipseのパッケージ・エクスプローラ上でダブルクリックして実行も出来ますし、 「リソースの検索(Ctrl + Shift + R)」で検索して選択しても実行できます。

設定ファイル

これらのタスクを実行するときに必要な設定ファイルがあります。これをビルドプロパティと呼びます。

ビルドプロパティは二種類の形式を持っています。 一部例外はありますが、基本的な区分けは以下の通りです。

「キー = 値」で表現される基本的なプロパティ
DBFluteクライアント配下の「./build-xxx.properties」：通称「properties」
「キー = ネスト構造(Map)」で表現されるプロパティ
DBFluteクライアント配下の「./dfprop/xxx.dfprop」：通称「dfprop」

dfpropの「ネスト構造(Map)」というのは、例えば以下のような形式を示します。 

ex) additionalForeignKeyMap.dfpropmap:{
    ; FK_SUMMARY_PRODUCT_PRODUCT_STATUS = map:{
        ; localTableName  = SUMMARY_PRODUCT     ; foreignTableName  = PRODUCT_STATUS
        ; localColumnName = PRODUCT_STATUS_CODE ; foreignColumnName  = PRODUCT_STATUS_CODE
    }
}

将来的に全てdfpropに移行される可能性があります。 無論、移行後も「properties」に定義されている設定で動かせるようにはします。

設定ファイルの具体的な内容は、ビルドプロパティをご覧下さい。

タスクの詳細

JDBC

自動生成のためのDBスキーマ情報の取得します。

DBFluteクライアントの「./dfprop/databaseInfoMap.dfprop」のDB接続情報からDBに接続し、 正常終了するとDBFluteクライアントの「./schema/project-schema-xxx.xml」にDBスキーマ情報が出力されます。

Generate

DBスキーマ情報からテーブル対応のクラス(+DBFluteフレームワーク)を自動生成します。

JDBCタスクが実行されていることが前提となります(DBスキーマ情報が必要なため)。 二回目以降の実行の場合は、変更のあったクラスだけが上書きされます。但し、Extended(ジェネレーションギャップ)のクラスは上書きされません。

クラスファイルは、「properties」に定義されている「torque.java.dir」プロパティに定義されたディレクトリ配下に出力されます。

Sql2Entity

外だしSQLからの検索結果を格納するEntity(CustomizeEntity)とSQLへの引数を格納するBean(ParameterBean)を自動生成します。

「properties」に定義されている「torque.java.dir」プロパティに定義されたディレクトリ配下のSQLファイルを対象とします。 Sql2Entityについて詳しくはこちら

クラスファイルは、「properties」に定義されている「torque.java.dir」プロパティに定義されたディレクトリ配下に出力されます。

ReplaceSchema

DBスキーマを初期化してDDL文を流しテストデータまで一気に登録します。

詳しくは、現場ソリューション２(PDF)をご覧下さい。

Doc

DBスキーマ情報をHTMLのテーブル一覧として出力します。

JDBCタスクが実行されていることが前提となります(DBスキーマ情報が必要なため)。

正常終了するとDBFluteクライアントの「./output/doc/project-schema-xxx.xml」にDBスキーマ情報が記載されたHTMLが出力されます。 サンプルをご覧下さい。

OutsideSqlTest

外だしSQLを2WaySQLとしてそのまま実行し、SQLの妥当性をチェックします。

詳しくは、現場ソリューション２(PDF)をご覧下さい。

ロールごとに必要なタスク

ロールごとに必要となるタスクが変わります。

  • アーキテクト : 全て(主にはJDBCとGenerate)
  • ディベロッパー : ReplaceSchemaとSql2Entity(OutsideSqlTestも可)

イベントごとに必要なタスク

イベントごとに必要となるタスクです。

  • DB作成時 : JDBC, Generate, Doc, (ReplaceSchema)
  • DB変更時 : JDBC, Generate, Doc, (ReplaceSchema)
  • 外だしSQL実装時 : Sql2Entity, (OutsideSqlTest)
  • 毎朝会社に来たら : ReplaceSchema
  • ナイトリービルド(テスト) : ReplaceSchema, OutsideSqlTest
