DBFluteの環境情報

製品ステータス

２００８年５月１７日時点で「0.7.1」のSandBox。

※但し、既に幾つもの開発現場で利用されています。

対応言語

Java-5.0 / Java-6.0 / C#-3.0

対応データベース

MySQL、PostgreSQL、Oracle、DB2、SQLServer、Apache Derby、H2 Database

対応DIコンテナ

Javaの場合Seasar-2.4.x(但しSpring-2.5.xでも動作可)。C#の場合Seasar.NET-1.3.7以降。

NGなテーブル設計

以下に当てはまるテーブル構造の場合は、残念ながらDBFluteを利用することができません。

  • テーブル名やカラム名にハイフン「-」やエスケープしないといけない特殊記号を利用している

非推奨なテーブル設計

以下に当てはまるテーブル設計の場合は、DBFluteの利用は可能ではありますが、一部機能に制限がある可能性があります。

  • 主キー制約が一切存在しない

    • → 主キーの無いテーブルに対してはUpdate/Deleteが出来ない(InsertはOK)。

  • FK制約が一切存在しない

    • → リレーションの定義をDBFluteの設定(AdditionalForeignKey)にて定義する必要あり。

  • ユニーク制約を付与できるところに付与されていない

    • → ConditionBeanで{親：子}のOneToOneリレーション(1:1)の結合ができない。

  • テーブル名やカラム名の命名ポリシーがバラバラ(ORDER_DETAILだったりOrderDetailだったり)

    • → その不統一性ががそのままプログラム上のクラス名やメソッド名にも反映される。

  • 更新日時や登録者などの共通カラムの命名ポリシーがバラバラ(UPDATE_DATEだったりUPD_DTTMだったり)

    • → DBFluteの「共通カラムの自動設定」の機能が利用できない。
